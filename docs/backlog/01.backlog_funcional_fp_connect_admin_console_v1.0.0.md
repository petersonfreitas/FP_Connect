# 1. Visão Geral

O **FP Connect Admin Console** será o painel administrativo central do ecossistema FPWebtech.

Ele será responsável por administrar empresas, usuários, permissões, sistemas contratados, planos básicos, status e auditoria do ecossistema.

Também será a **porta de entrada para todos os usuários e empresas**, permitindo que cada pessoa visualize e acesse, em nova janela, os sistemas aos quais possui permissão.

---

# 2. Objetivo do Produto

O objetivo do FP Connect Admin Console é permitir que a FPWebtech controle, de forma centralizada, quais empresas existem no ecossistema, quais sistemas elas contrataram, quem são seus usuários, quais permissões cada usuário possui e quais módulos estão ativos, suspensos ou em implantação.

O painel também deve funcionar como um portal de acesso aos sistemas contratados, respeitando o perfil do usuário.

---

# 3. Princípio Central

```text
O FP Connect Admin Console administra acesso, empresas, usuários, planos e permissões.
Cada sistema do ecossistema executa sua própria regra de negócio.
```

---

# 4. O Que Este Módulo Faz

O FP Connect Admin Console deve cuidar de:

```text
Cadastro de empresas/clientes
Gestão de usuários
Gestão de permissões
Liberação de sistemas por empresa
Status da empresa
Status dos sistemas contratados
Planos comerciais básicos
Portal de entrada para sistemas contratados
Auditoria administrativa
Visão administrativa do ecossistema
```

---

# 5. O Que Este Módulo Não Faz

Este módulo não deve executar regras específicas dos demais sistemas.

| Não deve fazer | Sistema responsável |
|---|---|
| Gerenciar oportunidades comerciais | CRM |
| Aprovar propostas comerciais | CRM |
| Calcular rota otimizada | Roteirizador |
| Rastrear entregas ou veículos em tempo real | Rotas/Tracking |
| Controlar MDF-e, pedágio e conciliações fiscais | EixoGuard |
| Criar produtos, cardápios e pedidos | Cardápio/Delivery |
| Processar eventos técnicos, conectores e retries | FP Connect IPaaS Operacional |
| Executar cobrança automática e faturamento completo | Billing |

# 8. Fluxos Funcionais

## 8.1 Fluxo — Cadastro de Empresa

```text
Super Admin acessa o Admin Console
↓
Seleciona "Nova Empresa"
↓
Informa dados básicos da empresa
↓
Define responsável principal
↓
Define status inicial
↓
Seleciona plano básico, se aplicável
↓
Seleciona sistemas contratados ou em implantação
↓
Salva cadastro
↓
Sistema registra auditoria
↓
Empresa passa a existir no ecossistema
```

### Critérios de aceite

```text
Somente usuário autorizado pode cadastrar empresa.
Empresa deve possuir pelo menos nome e responsável principal.
Empresa pode iniciar com status "em implantação".
Empresa pode iniciar sem todos os sistemas ativos.
Cadastro deve gerar auditoria.
```

---

## 8.2 Fluxo — Liberação de Sistema para Empresa

```text
Super Admin acessa cadastro da empresa
↓
Abre aba "Sistemas Contratados"
↓
Seleciona sistema do catálogo
↓
Define status: em implantação ou ativo
↓
Define observações, se necessário
↓
Confirma liberação
↓
Sistema passa a aparecer para usuários autorizados
↓
Ação é auditada
```

### Critérios de aceite

```text
Sistema não contratado não aparece para usuários da empresa.
Sistema em implantação pode aparecer com aviso.
Sistema ativo aparece como link de acesso.
Sistema suspenso bloqueia acesso.
Somente usuário autorizado pode liberar sistema.
Toda alteração deve ser auditada.
```

---

## 8.3 Fluxo — Cadastro de Usuário da Empresa

```text
Admin da Empresa ou Super Admin acessa "Usuários"
↓
Seleciona "Novo Usuário"
↓
Informa nome e e-mail
↓
Vincula usuário à empresa
↓
Seleciona sistemas permitidos
↓
Define papel por sistema
↓
Envia convite ou cria acesso
↓
Ação é auditada
```

### Critérios de aceite

```text
Admin da Empresa só cria usuários para a própria empresa.
Admin da Empresa só concede acesso a sistemas já contratados.
Admin da Empresa não concede permissão superior à sua.
Usuário pode ter acesso a um ou mais sistemas.
Usuário sem sistema liberado não visualiza atalhos operacionais.
Convite ou criação deve gerar registro de auditoria.
```

---

## 8.4 Fluxo — Portal de Entrada do Usuário

```text
Usuário acessa FP Connect Admin Console
↓
Realiza login
↓
Sistema identifica empresa(s) vinculada(s)
↓
Sistema identifica sistemas liberados
↓
Usuário visualiza tela "Meus Sistemas"
↓
Usuário clica em um sistema autorizado
↓
Sistema abre em nova janela
```

### Critérios de aceite

```text
Usuário comum visualiza apenas sistemas autorizados.
Sistemas devem abrir em nova janela.
Sistema suspenso não pode ser acessado.
Caso usuário tenha mais de uma empresa, deve selecionar o contexto correto.
Usuário sem sistemas disponíveis deve ver mensagem orientativa.
```

---

## 8.5 Fluxo — Suspensão de Empresa

```text
Super Admin acessa empresa
↓
Seleciona "Suspender Empresa"
↓
Informa motivo obrigatório
↓
Confirma suspensão
↓
Empresa perde acesso aos sistemas operacionais
↓
Usuários visualizam mensagem de bloqueio
↓
Ação é auditada
```

### Critérios de aceite

```text
Suspensão exige motivo.
Dados da empresa não são apagados.
Empresa suspensa não acessa sistemas operacionais.
Super Admin pode reativar empresa.
Reativação também deve ser auditada.
```

---

## 8.6 Fluxo — Inativação de Usuário

```text
Usuário autorizado acessa cadastro de usuários
↓
Seleciona usuário
↓
Informa motivo da inativação
↓
Confirma ação
↓
Usuário perde acesso ao ecossistema
↓
Ação é auditada
```

### Critérios de aceite

```text
Usuário inativo não pode logar ou acessar sistemas.
Inativação exige permissão adequada.
Inativação deve gerar auditoria.
Admin da Empresa só pode inativar usuários da própria empresa.
```

# 9. Regras de Negócio

## 9.1 Regras de Empresa

```text
Toda empresa deve ter status.
Toda empresa deve ter responsável principal.
Empresa em implantação pode ter acesso parcial.
Empresa ativa pode acessar sistemas contratados ativos.
Empresa suspensa não pode acessar sistemas operacionais.
Empresa cancelada não deve permitir novos acessos.
Dados de empresas suspensas ou canceladas não devem ser apagados automaticamente.
Cada empresa pode ter um ou mais sistemas contratados.
Cada sistema contratado pode ter status próprio.
```

---

## 9.2 Regras de Usuário

```text
Usuário pode estar vinculado a uma ou mais empresas.
Usuário pode ter papéis diferentes por sistema.
Usuário interno FPWebtech pode ter permissões globais.
Usuário de empresa deve estar vinculado a pelo menos uma empresa.
Usuário inativo não acessa nenhum sistema.
Usuário removido de uma empresa perde acessos vinculados a essa empresa.
Usuário não pode acessar sistema não contratado pela empresa.
Usuário comum visualiza apenas "Meus Sistemas".
```

---

## 9.3 Regras de Permissão

```text
Super Admin acessa tudo.
Admin FPWebtech acessa conforme permissões internas.
Admin da Empresa gerencia apenas a própria empresa.
Admin da Empresa pode criar usuários apenas da própria empresa.
Admin da Empresa só concede acesso a sistemas contratados.
Admin da Empresa não pode alterar plano.
Admin da Empresa não pode ativar sistema não contratado.
Usuário comum não gerencia permissões.
Permissão por sistema deve prevalecer sobre permissão genérica.
```

---

## 9.4 Regras de Sistemas Contratados

```text
Sistema globalmente disponível não significa contratado.
Sistema contratado pode estar em implantação, ativo, suspenso ou cancelado.
Sistema ativo aparece como atalho de acesso.
Sistema em implantação pode aparecer com aviso.
Sistema suspenso deve bloquear acesso.
Sistema cancelado deve ser preservado historicamente.
Alteração de status do sistema contratado deve ser auditada.
```

---

## 9.5 Regras de Portal de Entrada

```text
O Admin Console será a porta de entrada do ecossistema.
Sistemas contratados devem abrir em nova janela.
Super Admin e equipe FPWebtech visualizam painel completo.
Admin da Empresa visualiza versão limitada e links dos sistemas.
Usuário comum visualiza apenas tela simples de "Meus Sistemas".
Usuário sem acesso deve receber mensagem orientativa.
```

---

## 9.6 Regras de Planos Básicos

```text
Plano básico entra no MVP.
Plano básico não terá cobrança automática no MVP.
Plano pode agrupar sistemas inclusos.
Plano pode ser vinculado a uma empresa.
Alteração de plano deve ser auditada.
Billing completo será tratado em backlog próprio.
```

---

# 18. Definição Comercial Resumida

```text
O FP Connect Admin Console é o painel administrativo central da FPWebtech para gerenciar empresas, usuários, permissões, sistemas contratados, planos básicos e acesso aos produtos do ecossistema. Ele funciona como portal de entrada para todos os usuários, garantindo que cada empresa acesse apenas os sistemas contratados e que cada usuário visualize apenas os módulos autorizados.
```
