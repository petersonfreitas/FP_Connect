# 1. Visão Geral

O **FP Router** será o módulo futuro de roteirização inteligente, planejamento de rotas e apoio logístico/fiscal do ecossistema FPWebTech.

Ele deve absorver o conceito anteriormente tratado como **EixoGuard**, eliminando a necessidade de manter o EixoGuard como módulo separado.

O FP Router deve ser considerado **baixa prioridade** no momento atual, atuando como complemento futuro do **FP Tracking**.

---

# 2. Objetivo do Produto

O objetivo do FP Router é planejar rotas antes da execução operacional.

Ele deve receber entregas ou viagens pendentes, sugerir agrupamentos, organizar sequência de entrega, permitir aprovação humana e enviar o plano aprovado para o FP Tracking.

Em fase futura, também poderá apoiar conferências logísticas e fiscais relacionadas a rota, documentos e operação de transporte.

---

# 3. Princípio Central

```text
O FP Food é dono do pedido.
O FP Tracking é dono da execução e rastreamento.
O FP Router é dono do planejamento da rota.
O FP Fiscal é dono da emissão fiscal.
O FP Router pode consumir dados fiscais/logísticos, mas não substitui o FP Fiscal.
```

---

# 4. Decisão Arquitetural — EixoGuard

A partir desta versão:

```text
EixoGuard deixa de ser tratado como módulo independente.
O conceito EixoGuard passa a ser absorvido pelo FP Router.
```

Funções antes associadas ao EixoGuard devem ser reinterpretadas como recursos futuros do FP Router:

```text
Controle documental de viagem
Checklist logístico
Conferência fiscal/logística
Ocorrências de rota
Custos manuais de viagem
Pedágios e despesas
Motoristas e veículos associados a uma rota
```

---

# 5. Relação com Outros Sistemas do Ecossistema

## FP Tracking

Responsável por:

```text
Execução da entrega
Status da entrega
Localização em tempo real
Eventos de rota executada
Link público de rastreamento
Ocorrências operacionais de entrega
```

FP Router envia plano aprovado para FP Tracking.

## FP Food

Responsável por:

```text
Criar pedidos
Gerar entregas
Enviar entregas para FP Tracking
Consumir status de entrega
```

FP Food pode se beneficiar do FP Router no futuro para otimizar agrupamento de entregas.

## FP Fiscal

Responsável por:

```text
Emissão fiscal
Controle fiscal do pedido
Documentos fiscais de venda
Status fiscal associado ao pedido
```

FP Router pode consultar se uma entrega possui documentos fiscais necessários, mas não deve emitir documento fiscal do FP Food.

## FP Robots

Responsável por:

```text
Receber eventos do FP Router
Disparar alertas de rota aprovada, falha ou ocorrência
Enviar notificações
Automatizar integrações entre Router, Tracking e Food
```

## FP BI

Pode futuramente receber dados do FP Router para:

```text
Eficiência de rota
Tempo planejado versus executado
Custo por rota
Atrasos
Ocorrências logísticas
```

# 6. O Que o FP Router Faz

O FP Router deve cuidar de:

```text
Receber entregas pendentes
Agrupar entregas por região, horário ou critério operacional
Sugerir sequência de rota
Permitir ajuste manual da sequência
Permitir aprovação humana do plano
Enviar plano aprovado para FP Tracking
Registrar plano de rota
Comparar planejado versus executado, no futuro
Registrar checklist logístico
Registrar documentos associados à rota
Registrar ocorrências de planejamento
Registrar custos manuais de rota, no futuro
```

---

# 7. O Que o FP Router Não Faz

| Não deve fazer | Sistema responsável |
|---|---|
| Criar pedido | FP Food |

| Rastrear em tempo real | FP Tracking |
| Executar entrega | FP Tracking |
| Emitir documento fiscal de venda | FP Fiscal |
| Processar pagamento | FP Gateway |
| Enviar WhatsApp diretamente | FP Gateway, acionado por FP Robots |
| Decidir automações | FP Robots |
| Controlar cobrança SaaS | FP Billing |

# 10. Fluxo — Planejamento de Rota

```text
1. FP Tracking ou FP Food disponibiliza entregas pendentes.
2. FP Router lista entregas roteirizáveis.
3. Operador seleciona entregas.
4. FP Router sugere agrupamento e sequência.
5. Operador ajusta manualmente, se necessário.
6. Operador salva plano.
7. Plano fica aguardando aprovação, se configurado.
8. Admin aprova plano.
9. FP Router envia plano aprovado ao FP Tracking.
10. FP Tracking executa a operação.
11. FP Router recebe status consolidado ou consulta resultado posteriormente.
```

---

# 11. Fluxo — Checklist Logístico/Fiscal Futuro

```text
1. Plano de rota é criado.
2. FP Router identifica entregas/documentos associados.
3. FP Router exibe checklist logístico/fiscal.
4. Operador marca pendências resolvidas.
5. Se houver pendência crítica, rota pode ser bloqueada ou alertada.
6. FP Robots pode receber evento de pendência.
7. Após conferência, rota segue para aprovação.
```

# 16. Regras de Multiempresa

```text
Todo plano pertence a uma empresa.
Usuário só vê planos da empresa em que atua.
Entregas não podem ser misturadas entre empresas.
Super Admin pode visualizar globalmente conforme permissão.
Plano enviado ao Tracking deve preservar company_id.
```

---

# 17. Critérios Gerais de Aceite

```text
FP Router está formalizado como substituto futuro do EixoGuard.
EixoGuard não aparece mais como módulo independente.
FP Router recebe entregas pendentes e cria plano de rota.
Plano pode ser ajustado manualmente.
Plano aprovado pode ser enviado ao FP Tracking.
FP Tracking continua dono da execução.
FP Fiscal continua dono da emissão fiscal.
FP Router respeita multiempresa.
```

---

# 18. Riscos e Cuidados

| Risco | Mitigação |
|---|---|
| Duplicar funções do FP Tracking | Manter Router no planejamento e Tracking na execução |
| Tentar otimização avançada cedo demais | Começar com ajuste manual e aprovação humana |
| Confundir fiscal do Router com FP Fiscal | Router só consulta/valida documentos; FP Fiscal emite |
| Manter EixoGuard como módulo paralelo | Atualizar documentos e remover referência como módulo independente |
| Aumentar custo com mapas cedo demais | MVP pode usar roteirização simples/manual |

---

# 19. Conclusão

O **FP Router** deve ser mantido como módulo futuro de baixa prioridade.

Ele absorve o antigo conceito **EixoGuard** e se posiciona como complemento do **FP Tracking**, com foco em planejamento, roteirização inteligente futura e apoio logístico/fiscal, sem assumir a execução da entrega ou a emissão fiscal do FP Food.
